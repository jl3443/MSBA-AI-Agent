# 🔧 故障排除指南

## 快捷键不工作

### 问题：Command+Shift+B 打不开 Assistant

**解决方案：**

1. **检查扩展是否已加载**
   - 打开 `chrome://extensions/`
   - 确认 "UCLA MSBA AI Assistant" 已启用
   - 如果没有，点击"重新加载"按钮

2. **检查快捷键设置**
   - 访问 `chrome://extensions/shortcuts`
   - 找到 "UCLA MSBA AI Assistant"
   - 确认 "Toggle MSBA AI Assistant" 的快捷键是 `Command+Shift+B`
   - 如果没有设置，点击铅笔图标设置

3. **使用浮动按钮（最可靠）**
   - 打开任意网页
   - 查看右下角是否有蓝色的 🎓 按钮
   - 点击按钮打开 Assistant

4. **检查浏览器控制台**
   - 按 `F12` 打开开发者工具
   - 查看 Console 标签
   - 输入 `window.msbaAgent` 看是否有错误
   - 应该看到 MSBAAgent 对象

5. **重新加载页面**
   - 刷新当前网页（F5）
   - 快捷键可能需要页面重新加载才能生效

## 浮动按钮不显示

### 问题：看不到右下角的 🎓 按钮

**解决方案：**

1. **确认扩展已加载**
   - `chrome://extensions/` → 确认扩展已启用

2. **刷新网页**
   - 按 `F5` 刷新页面
   - 按钮应该在页面加载后出现

3. **检查是否有错误**
   - 按 `F12` 打开控制台
   - 查看是否有红色错误信息

4. **检查 API Key**
   - 点击扩展图标
   - 确认 API Key 已配置
   - 如果没有，先配置 API Key

## 其他问题

### Assistant 打开但没有响应

1. **检查 API Key**
   - 确认 API Key 正确
   - 确认有足够的 API 额度

2. **检查网络连接**
   - 确认能访问 api.deepseek.com 或 api.openai.com

3. **查看控制台错误**
   - 按 `F12` → Console
   - 查看具体错误信息

### 侧边栏显示但无法输入

1. **检查输入框是否存在**
   - 应该看到底部的输入框和"Send"按钮

2. **尝试点击输入框**
   - 确认可以输入文字

3. **检查 Send 按钮**
   - 点击 Send 按钮应该发送消息

---

## 快速测试步骤

1. ✅ 扩展已加载 (`chrome://extensions/`)
2. ✅ API Key 已配置（点击扩展图标检查）
3. ✅ 刷新网页（F5）
4. ✅ 查看右下角 🎓 按钮
5. ✅ 点击按钮或按 `Command+Shift+B`
6. ✅ 检查浏览器控制台（F12）是否有错误

如果以上都不行，请提供：
- 浏览器控制台的错误信息
- 扩展是否显示为已加载
- 是否能看到浮动按钮
